§ 00 — work method · file-based · tool-independent

Работа живёт в файлах, не в чате. Work lives in files, not in chat.

Файловый метод ведения работы. Две примитивы — arc (input → workspace → output) и goal (arc с целью, держащий вложенные арки). Состояние в файлах, а не в эфемерной переписке. Агенты, чаты, IDE — деталь реализации, не часть метода. Всё лежит в скрытом .arcs/ — код проекта не засоряется. A file-based method for running work. Two primitives — arc (input → workspace → output) and goal (an arc with a purpose holding nested arcs). State lives in files, not in an ephemeral chat. Agents, chats, IDEs are implementation details, not the method. Everything sits in a hidden .arcs/ — it never clutters project code.

§ 01

Суть. Две примитивы, состояние — это папка.The essence. Two primitives, state is a folder.

Чистая концепция, без инструментов. Метод не зависит от тулинга — CLI и скилл лишь обслуживают его.Pure concept, no tooling. The method doesn't depend on tools — the CLI and skill merely serve it.
arc

атом работыthe atom of work

Один законченный кусок. Папка NN-<title>/: input/ (что вошло в контекст) → workspace/ (вся работа в процессе) → output/ (самодостаточный результат — наружу читают только его). Плюс arc.md — паспорт: goal · status · указатели на output.One finished chunk. Folder NN-<title>/: input/ (what entered the context) → workspace/ (all work in progress) → output/ (a self-contained result — the outside reads only this). Plus arc.md — a passport: goal · status · output pointers.

goal

arc с цельюan arc with a purpose

Тот же скелет (input/ workspace/ output/) плюс собственный arcs/ и версионируемый статус-док NN-<slug>-goal.md. Для проекта или глубокого ресёрча — многоарочной работы.The same skeleton (input/ workspace/ output/) plus its own arcs/ and a versioned status doc NN-<slug>-goal.md. For a project or deep research — multi-arc work.

.arcs/

мета-директорияthe meta directory

Вся работа метода — в скрытом .arcs/ в корне проекта, чтобы не засорять код. arcs/ — отдельные арки, goals/ — многоарочные цели. Тулинг остаётся снаружи; проект получает только .arcs/.All method work lives in a hidden .arcs/ at the project root, so it never clutters the code. arcs/ — standalone arcs, goals/ — multi-arc goals. Tooling stays outside; a project gets only .arcs/.

Инкапсуляция — жёсткое правило. Всё, что нужно следующему шагу, должно быть выводимо из output/ в одиночку. workspace/ приватный, наружу не течёт. input/ — то, что пришло. Закрыл арку → output/ стоит сам по себе. Это OOP-аналогия: input = аргументы конструктора, workspace = приватное состояние, output = публичный интерфейс.Encapsulation is a hard rule. Anything the next step needs must be derivable from output/ alone. workspace/ is private and never leaks out. input/ is what came in. Close an arc → its output/ stands on its own. The OOP analogy: input = constructor args, workspace = private state, output = public interface.

Версионирование цели — номером, не правкой. Файлы 01-<slug>-goal.md, 02-…, 03-…. Высший номер = текущий = авторитетный; старые — история, их не трогают. Обновить цель = положить рядом новую версию. Версия фиксирует разворот.Versioning a goal — by number, not by edit. Files 01-<slug>-goal.md, 02-…, 03-…. The highest number = current = authoritative; old ones are history, never touched. Update a goal = drop a new version beside it. A version captures a pivot.

// автоматизируемоautomatable

Скрипт сканит arc.md/goal.md → статусы, открытые пункты, указатели. Это и есть arcs status — доска без отдельной базы.A script scans arc.md/goal.md → statuses, open items, pointers. That's exactly arcs status — a board with no separate database.

Инварианты. output/ не содержит факта, не обоснованного из input/ или процесса. workspace/ одноразовый — любой артефакт перегенерируем, финал в output/ не правят руками вне процесса. arc.md / NN-slug-goal.md — единственный источник статуса. Нумерация внутри одного arcs/ сплошная.Invariants. output/ holds no fact ungrounded in input/ or the process. workspace/ is disposable — any artifact is re-generable, the final in output/ is never hand-tweaked outside the process. arc.md / NN-slug-goal.md is the single source of status. Numbering is contiguous within one arcs/.

§ 03

Туллинг. Практический слой, отдельно от метода.Tooling. The practical layer, kept separate.

Метод живёт без инструментов. Но рядом есть CLI, скилл и хуки, чтобы не делать рутину руками — и чтобы агент не забывал про метод.The method lives without tools. But a CLI, a skill, and hooks sit alongside — so you don't do the chores by hand, and the agent doesn't forget the method.

Установка — клон + один инсталлер. install.sh кладёт arcs на PATH, линкует Claude-скилл и прописывает хуки. Метод-репо живёт где угодно (~/arcs); проект получает только .arcs/.Install — clone plus one installer. install.sh puts arcs on PATH, links the Claude skill, and wires the hooks. The method repo lives anywhere (~/arcs); a project gets only .arcs/.

# 1 · install (clone anywhere, then run the installer)
git clone https://github.com/socaseinpoint/arcs ~/arcs
cd ~/arcs && ./install.sh        # PATH + Claude skill + hooks
exec $SHELL                      # reload shell so `arcs` is found

# 2 · use it in a project
cd my-project && arcs init      # creates .arcs/ + asks the arc language
arcs new-goal payments
arcs new-arc -g payments stripe
arcs status

Апдейт — одна команда из любого места. CLI знает, куда клонировался: arcs update делает git pull метода и пере-проводит скилл и хуки (идемпотентно). Потом перезапусти сессию Claude Code.Update — one command, from anywhere. The CLI knows where it was cloned: arcs update does a git pull of the method and re-wires the skill and hooks (idempotent). Then restart your Claude Code session.

arcs update                    # git pull + re-wire PATH · skill · hooks

CLI arcs. Заводит скелет, считает нумерацию, читает доску. Команды совпадают с bin/arcs.The arcs CLI. Scaffolds the skeleton, computes numbering, reads the board. The commands match bin/arcs.

arcs init [en|ru|es]            # create .arcs/ + pick the arc language (asks if a TTY)
arcs lang [en|ru|es]            # show or change this project's arc language
arcs new-arc <slug>            # standalone arc in .arcs/arcs/
arcs new-arc -g <goal> <slug>   # an arc inside a goal
arcs new-goal <slug>           # a goal in .arcs/goals/
arcs status                    # the board: what's done, the bottleneck, the language
arcs update                    # git pull the method + re-wire skill/hooks

Язык — на проект. На arcs init ты выбираешь, на каком языке пишутся арки — en, ru или es; выбор лежит в .arcs/config и меняется в любой момент: arcs lang <code>. Шаблоны и текст, который генерят prose-агенты, следуют ему. Ключи полей (goal:, status:) остаются английскими — чтобы тулинг был языко-независим.Language — per project. At arcs init you pick the language arcs are written in — en, ru, or es; the choice lives in .arcs/config and changes anytime: arcs lang <code>. Templates and the prose the agents produce follow it. Field keys (goal:, status:) stay English — so the tooling is language-agnostic.

Агент (Claude, через скилл) держит содержимое арок на выбранном языке. Так можно вести весь проект в ru или es, не зная английского.The agent (Claude, via the skill) keeps arc content in the chosen language. So you can run a whole project's arcs in ru or es without knowing English.

cd my-project && arcs init ru    # arcs in this project are written in Russian
arcs lang es                  # switch to Spanish later (existing files unchanged)
§ 03.1

Три слоя принужденияThree enforcement layers

От слабого к сильному. Всё проводит install.sh. Проект без .arcs/ не задет — arcs init и есть опт-ин.Weakest to strongest. All wired by install.sh. A project without .arcs/ is never touched — arcs init is the opt-in.
// skill

Советует. Скилл подсказывает агенту использовать arcs — открыть арку и работать внутри неё (input → workspace → output), а не просто заметить папку.Advises. The skill nudges the agent to use arcs — to open an arc and work inside it (input → workspace → output), not merely notice the dir.

// arcs-hook

Напоминает. Хук на SessionStart и UserPromptSubmit инжектит напоминание и живую доску в проектах, которые опт-инулись — чтобы агент тихо не забыл.Reminds. A hook on SessionStart and UserPromptSubmit injects a reminder plus the live board in opted-in projects — so the agent can't quietly forget.

// arcs-gate

Блокирует. Хук на PreToolUse жёстко запрещает правки файлов проекта, пока не открыта арка. Правки внутри .arcs/ всегда разрешены; проекты без .arcs/ не затронуты.Hard-blocks. A PreToolUse hook refuses edits to project files until an arc is open. Edits inside .arcs/ are always allowed; projects without .arcs/ are never affected.

.arcs/ обычно коммитят — это память работы. Не хочешь — добавь .arcs/ в .gitignore. Нужны только bash + grep/sed (macOS/Linux), без шага сборки..arcs/ is usually committed — it's the memory of the work. If you'd rather not, add .arcs/ to .gitignore. Needs only bash + grep/sed (macOS/Linux), no build step.
