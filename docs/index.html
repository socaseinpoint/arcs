§ 00 — work method · file-based · agent-ready

Агент не теряет нить. Your agent never loses the thread.

arcs держит работу в файлах, а не в эфемерном чате — поэтому любой агент (или ты сам) продолжает ровно с того места, где остановился прошлый. Файловый метод: две примитивы — arc и goal, всё состояние в скрытой папке .arcs/. Работает и без агента — с ним выгода просто максимальная. arcs keeps work in files, not in an ephemeral chat — so any agent (or you) picks up exactly where the last one stopped. A file-based method: two primitives — arc and goal, all state in a hidden .arcs/ folder. Works without an agent too — it just pays off most with one.

§ 02

Старт за 30 секунд.Start in 30 seconds.

Клон + один инсталлер, потом arcs init в проекте. Дальше агент (через скилл) ведёт арки сам — CLI ты обычно не трогаешь.Clone plus one installer, then arcs init in a project. After that the agent (via the skill) runs the arcs for you — you rarely touch the CLI yourself.
# 1 · install (clone anywhere, then run the installer)
git clone https://github.com/socaseinpoint/arcs ~/arcs
cd ~/arcs && ./install.sh        # PATH + Claude skill + hooks
exec $SHELL                      # reload shell so `arcs` is found

# 2 · opt a project in — this is all you have to remember
cd my-project && arcs init      # creates .arcs/ + picks the arc language

# 3 · then just work. the agent opens arcs as it goes:
arcs new-goal payments         # multi-step thread → NN-@payments
arcs new-arc -g payments stripe # one step inside it
arcs candidate add-retries --from stripe  # park an idea that popped up → shows on the board
arcs close -g payments stripe   # done → writes output/, renames to __NN-stripe__
arcs status                    # the board — where everything stands

Без агента? Те же команды руками — метод не зависит от тулинга. С агентом? Поставил скилл + хуки и забыл: агент сам открывает арку перед работой и держит output/ чистым. Апдейт — arcs update из любого места.No agent? Run the same commands by hand — the method doesn't depend on tooling. With an agent? Install the skill + hooks and forget it: the agent opens an arc before working and keeps output/ clean. Update with arcs update from anywhere.

§ 03

Как устроено. Две примитивы, состояние — это папка.How it works. Two primitives, state is a folder.

Чистая концепция. Метод не зависит от инструментов — CLI и скилл лишь обслуживают его.The pure concept. The method doesn't depend on tools — the CLI and skill merely serve it.
arc

атом работыthe atom of work

Один законченный кусок. Папка NN-<title>/: input/ (что вошло в контекст) → workspace/ (вся работа в процессе) → output/ (самодостаточный результат — наружу читают только его). Плюс arc.md — паспорт: goal · status · указатели.One finished chunk. Folder NN-<title>/: input/ (what entered the context) → workspace/ (all work in progress) → output/ (a self-contained result — the outside reads only this). Plus arc.md — a passport: goal · status · pointers.

goal

arc с цельюan arc with a purpose

Тот же скелет (input/ workspace/ output/) плюс собственный arcs/ и версионируемый статус-док NN-<slug>-goal.md. Для проекта или глубокого ресёрча — многоарочной работы.The same skeleton (input/ workspace/ output/) plus its own arcs/ and a versioned status doc NN-<slug>-goal.md. For a project or deep research — multi-arc work.

.arcs/

мета-директорияthe meta directory

Вся работа метода — в скрытом .arcs/ в корне проекта, чтобы не засорять код: config, rules/ и единый поток arcs/. В нём общая сквозная нумерация: арка — NN-<slug>/, цель — NN-@<slug>/ (@ метит цель). Отдельной goals/ больше нет. Тулинг снаружи; проект получает только .arcs/.All method work lives in a hidden .arcs/ at the project root, so it never clutters the code: config, rules/ and one single stream arcs/. It carries shared continuous numbering: an arc is NN-<slug>/, a goal is NN-@<slug>/ (the @ marks a goal). There's no separate goals/ anymore. Tooling stays outside; a project gets only .arcs/.

Инкапсуляция — жёсткое правило. Всё, что нужно следующему шагу, должно быть выводимо из output/ в одиночку. workspace/ приватный, наружу не течёт. input/ — то, что пришло. Закрыл арку → output/ стоит сам по себе. OOP-аналогия: input = аргументы конструктора, workspace = приватное состояние, output = публичный интерфейс.Encapsulation is a hard rule. Anything the next step needs must be derivable from output/ alone. workspace/ is private and never leaks. input/ is what came in. Close an arc → its output/ stands on its own. The OOP analogy: input = constructor args, workspace = private state, output = public interface.

Версионирование цели — номером, не правкой. Файлы 01-<slug>-goal.md, 02-…, 03-…. Высший номер = текущий = авторитетный; старые — история, их не трогают. Обновить цель = положить рядом новую версию. Версия фиксирует разворот.Versioning a goal — by number, not by edit. Files 01-<slug>-goal.md, 02-…, 03-…. The highest number = current = authoritative; old ones are history, never touched. Update a goal = drop a new version beside it. A version captures a pivot.

§ 04

С агентом — или без.With an agent — or without.

Метод стоит сам по себе: папки плюс правило инкапсуляции. Агент, скилл и хуки делают так, что метод соблюдается сам — от слабого принуждения к жёсткому.The method stands on its own: folders plus the encapsulation rule. An agent, a skill, and hooks make it enforce itself — from soft nudge to hard block.
// skill

Советует. Скилл подсказывает агенту использовать arcs — открыть арку и работать внутри неё (input → workspace → output), а не просто заметить папку.Advises. The skill nudges the agent to use arcs — open an arc and work inside it (input → workspace → output), not merely notice the dir.

// arcs-hook

Напоминает. Хук на SessionStart и UserPromptSubmit инжектит напоминание и живую доску в опт-инутых проектах — чтобы агент тихо не забыл.Reminds. A hook on SessionStart and UserPromptSubmit injects a reminder plus the live board in opted-in projects — so the agent can't quietly forget.

// arcs-gate

Блокирует. Хук на PreToolUse жёстко запрещает правки файлов проекта, пока не открыта арка. Правки внутри .arcs/ всегда разрешены; проекты без .arcs/ не затронуты.Hard-blocks. A PreToolUse hook refuses edits to project files until an arc is open. Edits inside .arcs/ are always allowed; projects without .arcs/ are never affected.

Без агента arcs — это дисциплина. Ты сам заводишь папки и держишь output/ самодостаточным. Ценность — структура и «ты здесь», даже в одиночку.Without an agent, arcs is a discipline. You create the folders and keep output/ self-contained yourself. The value — structure and "you are here" — holds even solo.

С агентом arcs соблюдается само. Скилл + хуки превращают метод из «надо помнить» в «нельзя обойти» — и тут максимальная выгода: холодный хэндофф между сессиями и агентами. Поведение настраивается правилами: тела глобальны в метод-репо (приезжают с arcs update), включаются на проект в .arcs/config — subagents on по умолчанию: делегировать каждую арку сабагенту, независимые параллельно. Язык арок — на проект (arcs lang en|ru|es), так что весь проект можно вести на русском. .arcs/ обычно коммитят — это память работы. Нужны только bash + grep/sed, без шага сборки.With an agent, arcs enforces itself. The skill + hooks turn the method from "must remember" into "can't skip" — and that's where the payoff peaks: cold handoff across sessions and agents. Behavior is tuned by rules: bodies are global in the method repo (they arrive with arcs update), switched on per project in .arcs/config — subagents on by default: delegate each arc to a subagent, independent ones in parallel. The arc language is per project (arcs lang en|ru|es), so a whole project can run in Russian or Spanish. .arcs/ is usually committed — it's the memory of the work. Needs only bash + grep/sed, no build step.
